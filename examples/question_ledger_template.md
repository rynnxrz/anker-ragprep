# 问题台账模板（Question Ledger）

> 用途：为研究推进模式下的 P1 问题提供结构化暂存，避免问题散落在对话里。
>
> 运行时约定：每轮只展示**新增 / 变更**的行，历史行压缩为一句 `见完整台账 (N 条已归档)`；用户要完整台账时才全量打印。
>
> 存放路径：case 产出目录下 `output/<case_id>/question_ledger.md`。

---

## 字段定义

| 字段 | 说明 | 取值 |
|------|------|------|
| time | 首次提出的时间戳 | `YYYY-MM-DD HH:MM` |
| question | 问题本身，一句话 | 中文 |
| level | 问题级别 | `P0 / P1 / P2` |
| impact | 不回答会影响什么决策 | 一句话 |
| assumption_if_no_answer | 不回答时的默认假设 | 一句话 |
| deadline | 答复截止时间 | `YYYY-MM-DD` |
| status | 状态 | `open / answered / assumed / closed` |

字段顺序与列数固定；用 `|` 分隔，TSV 友好，可直接粘贴到 Excel。

## 使用规则

1. **新建**：模型生成 L1 时，每发现一个 P1 就追加一行，级别字段写 `P1`。P2 直接进 L1 `默认假设` 段，不进台账。P0 立刻提问，问完标 `answered` 或 `closed`。
2. **更新**：用户回答后，在同一行把 `status` 从 `open` 改为 `answered`，并把答复附在 `question` 后面（用 `→ <answer>`）。
3. **提问触发**：`level=P1 AND status=open` 的条数 `>= 3` 时，模型在下一次 L1 输出中把这 3 条合并为一个 `不确定的问题` 批次一次性提问（遵守 R-ASK-002）。
4. **归档**：`status=closed` 的行保留不删，但运行时展示时折叠为一句 `见完整台账 (N 条已归档)`。
