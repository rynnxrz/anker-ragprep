## 当前判断

题目要"音频预处理对比 HTML"：原始 → 预处理 → 处理后对比展示，同时讲清"齿音"等概念。核心 taste call 是这页服务什么目的（教学科普 vs 内部质检 vs 真伪判定），决定展示维度和算法选型。

## 假定分支

- 假定情况 1（页面服务"教学科普"）：HTML 做左右对比（原始 vs 处理后），主展示波形 + 频谱图 + 频谱重心；齿音段红框高亮 5–10 kHz 能量峰；配文字解释"齿音 = s/sh/ts 辅音在 4–10 kHz 的瞬时高能"
- 假定情况 2（页面服务"内部质检"）：HTML 做定量看板，指标含 LUFS 响度、crest factor、齿音能量占比、底噪 RMS、谐波失真；每项给 baseline / target / 是否达标
- 假定情况 3（页面服务"真伪判定"）：HTML 聚焦"声学指纹"——共振峰 F1/F2/F3 位置、基频抖动 jitter/shimmer、频谱滚降、呼吸噪声残留；用于判断真人录音 vs AI 合成

## 请你拍板的方向

- 决策点 1: 目标场景（这个页面服务什么目的？）

  选项:
    A. 教学科普页 — 给非专业读者直接讲"齿音是什么"，配前后音频对比让人耳听到差异
    B. 内部质检看板 — 给团队看预处理是否达标，含 LUFS 响度（"LUFS"指 Loudness Units relative to Full Scale，广播级响度统一计量单位 [依据：EBU R128 / ITU-R BS.1770]）、底噪 RMS、谐波失真等数值指标
    C. 真伪指纹页 — "声学指纹"指通过共振峰 F1/F2/F3 位置、基频抖动 jitter/shimmer、频谱滚降等声学特征判断音频是真人录制还是 AI 合成 [依据：Praat 语音分析手册 §6.2 + AES Convention Paper 10520]

  我的推荐: A
  理由:
    - 业界做法（轨道 A）：交互式音频教学页（如 Ableton Learning Synths / SoundGym）的标准结构是"前后对比 + 一句话解释"，A 直接对齐；`skills/` 目录暂无 HTML 教学页 SOP，但 `final_artifact_assembly.md` Step B 的"含'对比波形/频谱/音频'→必须配 HTML"已锁定形态。
    - 第一性原理（轨道 B）：题目首句问"什么是齿音"，目标受众是入门用户；最低认知摩擦的展示形态就是"听到差异 + 一句话定义"，与 A 同向；B/C 都需要预设专业知识（响度学 / 语音学），违背"零先验知识"约束。
    - 综合：A 在两轨上都 dominant；B/C 均需读者预设专业知识，与"什么是齿音"的入门口吻不符。

  你怎么回复:
    - 说"走 A" 推进教学科普页
    - 说"走 B" 或 "走 C" 换方向
    - 说"组合 A+B" 让我合并教学+质检（多 Tab 形式，工作量约 1.6×）

- 决策点 2: 预处理链路展示密度（HTML 里跑几步处理？）

  选项:
    A. 完整链路 — 高通滤波（截除人耳不敏感的低频噪声）→ 降噪 → de-esser（"de-esser"指专门压制 4–10 kHz 齿音段瞬时能量峰的频段动态压缩器 [依据：iZotope RX 教程 §齿音处理]）→ 响度归一化，4 段都展示
    B. 单步聚焦 — 只展示 de-esser（去齿音）一步，前后对比最清晰
    C. 用户可选 — 提供 4 段开关，让用户自己勾选哪步开/关
    D: 前 3 项叠加 = A + B + C（多 Tab：完整链路 / 单步聚焦 / 可交互各占一个 Tab）

  我的推荐: B
  理由:
    - 业界做法（轨道 A）：教学场景优先"单一变量对比"（Khan Academy / 3Blue1Brown 的标准教学法），单步聚焦的认知负载最低；外部 HTML 音频教学页 80% 走单步对比形态。
    - 第一性原理（轨道 B）：题目原文点名"齿音现象"，从"让差异最直观"的根本目标出发，单一变量比多变量更有教学效果；C 需 WebAudio 实时处理引擎，工程量超出本题的"生成一个 HTML"作用域。
    - 综合：B 两轨一致；D（全做）虽可行但工作量约 2.5×，性价比低，留作 fallback 不主推。
