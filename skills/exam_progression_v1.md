# Skill: Exam Progression v1（多轮考试节奏 SOP）

> 目的：把 3 小时多轮考试拆成可执行的三阶段节奏。每个阶段定义**输入 / 输出 / 退出条件 / 推进 vs 等待 判据**，让模型每轮知道自己应该停在哪个 phase、什么时候推进、什么时候等用户。
>
> 触发条件：每个 case 第一轮自动进入 Phase 1；后续轮次按用户消息特征自动迁移 phase。

---

## 三阶段总览

| Phase | 名称 | 典型轮次 | 主输出形态 | 退出条件 |
|-------|------|---------|----------|---------|
| Phase 1 | 题目消化 + 切入角度选择 | 第 1-2 轮 | L1（含切入角度类的"请你拍板的方向"） | 用户对切入角度有定调 |
| Phase 2 | 迭代深化 | 中间 N 轮 | L1（缩窄维度，深化某一分支） | 主方案分支收敛到 1 个 + 关键决策点已拍板 |
| Phase 3 | 交付物组装 | 最后 1 轮 | L3 完整 .md（必要时配 HTML） | 用户接受最终交付物 |

---

## Phase 1：题目消化 + 切入角度选择

### 输入信号
- 用户第一条消息粘贴题目原文，或重启会话粘贴新题目。

### 模型该做的
1. 把题目原文当作**唯一信息源**消化。**禁止**向用户索要更多背景（信息缺口走 Failure Fallback：代理指标 + 假设继续）。
2. 内部完整跑 ACVD（Axioms / Constraints / Variables），识别 2-4 个**可能的切入角度**（如：从用户洞察切入 / 从技术可行性切入 / 从竞品对标切入 / 从单位经济模型切入）。
3. 对外只输出 L1 6 段。在 `假定分支` 段把 2-4 个切入角度对应的方案并列；在 `请你拍板的方向` 段**至少放一条"切入角度选择"**类的决策点。

### 退出条件（迁移到 Phase 2）
满足以下任一即可：
- 用户明确说"走假定情况 N" / "选 A" / "用第一个切入角度"
- 用户反驳"都不对，应该 X"（接受用户提出的新切入）
- 用户给出新约束/新数据让某个分支自动出局

### 不该做的
- ❌ 在第一轮就跳到 L3 全量 ACVD 输出（违反渐进披露）
- ❌ 在第一轮 `请你拍板的方向` 里只问"补数据"类问题（应至少有一条方法论/角度选择类）
- ❌ 输出末尾追加规则自检 8 条（违反 R-CHECK-001 默认隐藏）

---

## Phase 2：迭代深化

### 输入信号
- 用户对某分支 / 方向作出选择
- 用户补充新约束 / 新数据 / 新偏好
- 用户对模型输出某一段提出反驳或追问

### 模型该做的
1. **重跑相关的 ACVD 子树**（不重跑全量），更新受影响的 axioms / variables。
2. 输出新一轮 L1，**缩窄到用户选定的分支**：
   - `当前判断` 反映已收敛的方向
   - `假定分支` 列出该方向下更细的子分歧（2-4 条）
   - `请你拍板的方向` 给出该方向下的下一层决策点
3. 在 `默认假设` 段明确标记"已用户拍板"的部分（如：`切入角度=齿音物理特征 [用户已拍板]`），与待假设的 `⚠ assumed` 区分。

### 退出条件（迁移到 Phase 3）
满足**全部**条件：
- 主方案路径已收敛到 1 个（无并行分支）
- `请你拍板的方向` 段在最近一轮为空 / 仅剩 1 条且已用户回应
- Decision Ledger 内部 ≥ 3 条草稿
- 用户**显式触发**：`/finalize` / `做最终交付` / `给我最终交付物` / `给我考试 final` 之一

### 推进 vs 等待 的判据

| 用户输入特征 | 模型动作 |
|------------|---------|
| 明确选了一个分支 | 推进，不复述用户选择，直接缩窄到该分支输出新 L1 |
| 提出新约束 / 新数据 | 重跑受影响的 ACVD 子树，更新 L1 中受影响的段 |
| 问"为什么是 X 不是 Y" | 给 L2 摘要（关键行级别），不展开 L3 |
| 长时间无新输入 | 不主动追加，等用户开口 |
| 反驳整个方向 | 回退到 Phase 1，重新出切入角度分支 |

### 不该做的
- ❌ 每轮都重新输出完整 ACVD（应只更新受影响部分）
- ❌ 复述用户选择（"好的，我理解您选择了..."）—— 直接推进
- ❌ 在收敛中途擅自跳到 Phase 3（必须等用户触发 finalize）

---

## Phase 3：交付物组装

### 输入信号
- 用户消息含 `/finalize` / `做最终交付` / `给我最终交付物` / `给我考试 final` / `完整版` / `L3` / `full report` 之一

### 模型该做的
调用 `skills/final_artifact_assembly.md` 的多轮收敛 SOP，输出：
1. 完整 STEP-1..STEP-6 的 .md（含 ACVD + Decision Ledger + 三张表）
2. 必要时配套 HTML（参考 `examples/golden_case.html` 结构）

HTML 触发判据（满足任一即输出 HTML）：
- 题目原文明确要求"生成 HTML" / "做一个网页" / "可视化对比"
- 用户在多轮中显式说"要 HTML"
- 题目涉及波形 / 频谱 / 流程图 / 数据看板等可视化对比场景

### 退出条件
- 用户接受交付物（无新追加要求），或要求微调（回到局部 L2 修改，不重启 Phase 1）

---

## Phase 自动识别（无需用户标注）

模型每轮开头先内部判断当前 phase：

```
输入特征判断：
- 是否首轮 / 是否新题目？ → Phase 1
- 用户是否含 finalize 触发词？ → Phase 3
- 否则 → Phase 2
```

判断完成后，按对应 phase 的"模型该做的"清单执行。
