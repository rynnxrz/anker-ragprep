# 规则契约 v1（可检查）

## 目的
将输出风格约束从“临时聊天偏好”升级为“仓库内可执行规则”。

## Rule IDs

### R-ASK-001
- 名称：提问题禁止总结与比喻
- 要求：提问时只允许“问题位置 + 问题本身”。
- 禁止：先做总结、引入类比/隐喻、先给宏观概述再提问。
- 合格示例：
  - 位置：第 3 段第 2 句；问题：这里将“找对人”和“分好钱”并列，但未给出衡量标准。

### R-SUM-001
- 名称：总结禁止比喻与比喻式改写
- 要求：总结直接陈述事实与判断。
- 禁止：使用比喻、借物喻人、用“换一种说法”进行比喻式重写。

### R-CODE-001
- 名称：代码修改请求只输出完整可执行命令
- 要求：涉及代码改动时，先内部形成思路，最终仅输出完整命令（可多条）。
- 禁止：输出完整代码块作为最终交付。

### R-LANG-001
- 名称：L1 输出必须使用中文标签
- 要求：必须使用 `当前判断` / `下一步动作` / `不确定的问题` / `默认假设` / `证据` / `规则自检`。
- 禁止：使用 `Now` / `Next` / `Blockers` / `Blocker` / `Assumptions` / `Assumption` / `Evidence` / `Self-check`，以及这些词的任何中英混写变体（如 `Blockers（P0/P1）` / `## 不确定的问题（Blockers）` / `Blockers（≤3）`）。
- 例外：仅在用户一次性显式说“用英文输出 / English output”时才允许切换英文标签；下一次对话自动回落中文默认。

### R-FMT-001
- 名称：L1 输出禁止 meta 标题
- 要求：L1 输出第一行必须是正文第一个中文段标题（即 `## 当前判断`）。
- 禁止：在顶部加任何 meta 框架标题，包括但不限于 `L1 决策卡 — <case 名>` / `L1 xxx` / `L2 xxx` / `L3 xxx` / `<任意>决策卡` / `决策 card` / `轻量卡片` / `分析卡`。
- 原因：这些词是对“答题”这件事的比喻式包装，既违反 R-SUM-001（禁比喻），也暴露了内部分层标签（L1/L2/L3）给审阅者。
- 替代：若确需题目回显，放在 `当前判断` 首句以事实陈述方式说明，而不是作为独立标题行。

### R-CHECK-001
- 名称：强制规则自检
- 要求：每次回答末尾追加“规则自检”段落，逐条标注是否违反：
  - R-ASK-001: Yes/No
  - R-ASK-002: Yes/No
  - R-LANG-001: Yes/No
  - R-FMT-001: Yes/No
  - R-SUM-001: Yes/No
  - R-CODE-001: Yes/No（无代码场景标 N/A）

## 冲突处理
当用户临时要求与本契约冲突时，优先遵守本契约。
