# 规则契约 v1（可检查）

## 目的
将输出风格约束从“临时聊天偏好”升级为“仓库内可执行规则”。

## Rule IDs

### R-ASK-001
- 名称：提问题禁止总结与比喻
- 要求：提问时只允许“问题位置 + 问题本身”。
- 禁止：先做总结、引入类比/隐喻、先给宏观概述再提问。
- 合格示例：
  - 位置：第 3 段第 2 句；问题：这里将“找对人”和“分好钱”并列，但未给出衡量标准。

### R-SUM-001
- 名称：总结禁止比喻与比喻式改写
- 要求：总结直接陈述事实与判断。
- 禁止：使用比喻、借物喻人、用“换一种说法”进行比喻式重写。

### R-CODE-001
- 名称：代码修改请求只输出完整可执行命令
- 要求：涉及代码改动时，先内部形成思路，最终仅输出完整命令（可多条）。
- 禁止：输出完整代码块作为最终交付。

### R-CHECK-001
- 名称：强制规则自检
- 要求：每次回答末尾追加“规则自检”段落，逐条标注是否违反：
  - R-ASK-001: Yes/No
  - R-SUM-001: Yes/No
  - R-CODE-001: Yes/No

## 冲突处理
当用户临时要求与本契约冲突时，优先遵守本契约。
